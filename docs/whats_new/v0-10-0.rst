v0.10.0 - Under development
+++++++++++++++++++++++++++

API Changes
###########
- Passing per-quantity unit keyword arguments to
  :py:meth:`~tespy.networks.network.Network.set_attr` (e.g.
  :code:`T_unit`, :code:`p_unit`, :code:`h_unit`, ...) now raises a
  :code:`TypeError`. Use
  :py:meth:`~tespy.tools.units.Units.set_defaults` instead, e.g.
  :code:`nw.units.set_defaults(temperature='degC', pressure='bar')`.
  The shorthand :code:`"C"` for Celsius is no longer accepted; use
  :code:`"degC"` or :code:`"°C"` instead
  (`PR #968 <https://github.com/oemof/tespy/pull/968>`__).
- The :code:`power_connector_location` attribute of :code:`SimpleHeatExchanger`
  is deprecated and no longer required. Use :code:`HeatConnection` together
  with the new :code:`HeatSource` and :code:`HeatSink` components instead -
  the inlet or outlet side is detected automatically from the connection
  (`PR #961 <https://github.com/oemof/tespy/pull/961>`__).
- The :code:`Bus` class has been removed. Use :code:`PowerConnection` together
  with :code:`PowerBus`, :code:`Motor` and :code:`Generator` instead
  (`PR #964 <https://github.com/oemof/tespy/pull/964>`__).
- The built-in :code:`ExergyAnalysis` class has been removed and its
  functionality moved to the external library
  `exerpy <https://github.com/oemof/exerpy>`__. Use
  :py:meth:`tespy.models.template.ModelTemplate.run_exergy_analysis` or call
  :code:`ExergyAnalysis.from_tespy(nw, Tamb, pamb)` directly from exerpy
  (`PR #963 <https://github.com/oemof/tespy/pull/963>`__).

Other Changes
#############
- Installation instructions now recommend
  `uv <https://docs.astral.sh/uv/>`__ as the primary package manager
  (`PR #967 <https://github.com/oemof/tespy/pull/967>`__).
- :py:meth:`~tespy.networks.network.Network.get_ude` has been added,
  analogous to :py:meth:`~tespy.networks.network.Network.get_comp` and
  :py:meth:`~tespy.networks.network.Network.get_conn`, to retrieve a
  :code:`UserDefinedEquation` by label. Not found labels currently return
  :code:`None`, they will raise a :code:`KeyError` in a future version
  (`PR #966 <https://github.com/oemof/tespy/pull/966>`__).
- :py:meth:`~tespy.networks.network.Network.save` now accepts an
  :code:`as_dict=True` flag to return the network state as a plain
  :code:`dict`, which can be passed directly as :code:`design_path` or
  :code:`init_path` in a subsequent :meth:`~tespy.networks.network.Network.solve`
  call without writing to disk. Passing :code:`None` without :code:`as_dict=True`
  still works but is deprecated
  (`PR #962 <https://github.com/oemof/tespy/pull/962>`__).
- Mixing rules are now registered via a :code:`MixingRuleRegistry`, making it
  possible to add custom mixing rules at runtime
  (`PR #960 <https://github.com/oemof/tespy/pull/960>`__).
- You can now pass a :code:`postproc_func` argument to
  :py:meth:`~tespy.models.template.ModelTemplate.sensitivity_analysis`, which
  accepts any callable :code:`postproc_func(model) -> dict | None` to be
  executed after each solve step
  (`PR #963 <https://github.com/oemof/tespy/pull/963>`__).
- The automatic y-axis limits of
  :py:meth:`~tespy.models.template.ModelTemplate.plot_Ts_diagram_matplotlib`
  and
  :py:meth:`~tespy.models.template.ModelTemplate.plot_logph_diagram_matplotlib`
  now extend to at least the fluid's critical temperature or critical pressure
  respectively, so the two-phase dome is always fully visible when no explicit
  :code:`ylim` is passed
  (`PR #963 <https://github.com/oemof/tespy/pull/963>`__).

Bug Fixes
#########
- :py:meth:`~tespy.networks.network.Network.export` and
  :py:meth:`~tespy.networks.network.Network.save` now create the target
  directory if it does not exist yet
  (`PR #965 <https://github.com/oemof/tespy/pull/965>`__).
- :py:meth:`~tespy.networks.network.Network.save_csv` no longer writes
  :code:`Component` CSV files inside the :code:`Connection` subdirectory
  when both are exported together
  (`PR #966 <https://github.com/oemof/tespy/pull/966>`__).
